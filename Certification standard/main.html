This document defines the specification for certification of smart contracts on the Cardano blockchain. It outlines three different levels of certification which can be used to show that a smart contract went has been reviewed and was found to behave as expected and to not be vulnerable to a set of security weaknesses.
This is a draft for the first version of the Cardano Certification Level Standards. The Working Group meets every two weeks on [Discord](https://discord.com/channels/1029837832350605343/1030116075532206101). You can send questions on this [Discord Channel](https://discord.com/channels/1029837832350605343/1030116075532206101) or as [GitHub Issues]("https://github.com/input-output-hk/Certification-working-group"). What is certified?

A certification refers to a DApp, which means the source code of the on-chain component of a smart contract or several smart contracts, along the UPLC compiled from the source code and a given parametrisation. Certification Levels

Three certification levels are defined, named 1, 2, and 3. [TBC: We have a chance to rename, redefine, reorganize, renumber the levels, etc.]

Level 1: Automated testing

Certification at level 1 is intended for verification of a DApp that can be performed by an unguided automated tool. The tool shall be able to determine if the code meets its requirements and is not vulnerable to some vulnerabilities [Discussion: Should we relate to the list of vulnerabilities and define the ones that can/should be dealt at this level, because the tools can do it]. Level 2: Manual audit

Certification at level 2 is intended for a DApp that has been carefully reviewed by an auditor, through a manual analysis with possibly the use of automated tools. [Discussion: How de we split this from auditor doing only the work of level 1?] Say something about CIP 52.

Level 3: Formal verification

Certification at level 3 is intended for the verification of a DApp that went through formal verification.
